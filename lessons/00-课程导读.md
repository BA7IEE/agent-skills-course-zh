# 00 课程导读

## 这章讲什么

这一章先不急着讲技术细节，而是先把整套课的地图铺开：

- 这套课到底在教什么
- 为什么 `Agent Skills` 值得学
- 学完之后你能做什么
- 这套中文整理版应该怎么读

## 你可以先带着这个问题来读

如果你现在对 `Agent Skills` 还没有概念，可以先只想一个最现实的问题：

> 为什么有些人用 AI 时越来越顺，有些人却总在重复解释同一件事？

这套课本质上就是在回答这个问题。

## 先解释两个词

- `Agent`：可以理解为“为了完成目标，会自己调用工具、拆分任务、继续推进的 AI 助手”
- `Skill`：可以理解为“教这个 AI 在某类任务里按固定方法做事的一套说明和资源”

## 用大白话先讲明白

很多人用 AI 助手时，最大的痛点不是“模型不聪明”，而是“每次都要重新解释一遍我要它怎么做”。

比如你可能反复让 AI：

- 按某个固定模板分析表格
- 按某个固定口径审代码
- 按某个固定结构写报告

如果这些事情每次都靠临时写 prompt，不仅麻烦，还不稳定。

`Skill` 的价值就在这里：它把一套反复会用到的工作方法打包起来，让 agent 在遇到对应任务时，知道该怎么做、去哪看资料、什么时候运行脚本。

## 原课主线

原课程围绕一个核心问题展开：

> 怎样把“重复解释一次又一次”的工作方式，升级成“可复用、可迁移、可维护”的 agent 工作流？

它会带你依次理解：

- skills 的基本概念
- skills 为什么适合做成开放标准
- skills 和 tools、MCP、subagents 的边界
- 如何创建自己的 skills
- 如何在 `Claude.ai`、`Claude API`、`Claude Code`、`Agent SDK` 中落地

## 这一章和后面几章是什么关系

你可以把这套课理解成三段：

1. 前 3 章先建立概念框架
2. 中间几章讲 skill 怎么看、怎么写
3. 最后几章讲不同环境里怎么落地

所以本章的作用，不是让你学会一个具体技巧，而是让你先知道整张地图长什么样。

## 这套中文整理版和原课有什么不同

和原课相比，这个版本更重视三件事：

### 1. 先讲人话，再讲术语

原课默认你能比较自然地跟着英文术语走。中文读者往往不是不会学，而是会被一堆名词卡住。所以这里会先解释“这个东西到底解决什么问题”，再告诉你它的正式名字。

### 2. 更照顾零基础读者

原课里很多内容默认你知道：

- Claude.ai 和 API 有什么区别
- 文件系统和 bash 为什么会影响 skill 的可用性
- 什么叫开放标准

中文整理版会把这些断点补上。

### 3. 更适合 GitHub 阅读

原课程是视频课逻辑，本仓库改造成了文档课逻辑。你可以：

- 从头顺序学
- 按章节跳读
- 先看术语表再回来继续

## 学习建议

如果你完全是第一次接触这个主题，建议这样读：

1. 先读本章
2. 再读第 01、02、03 章，把概念边界搞清楚
3. 接着读第 05 章，理解一个 skill 到底怎么写
4. 最后按需看 API、Claude Code、Agent SDK 章节

## 中文场景例子

如果你现在还觉得这个主题有点抽象，可以先想一个很常见的中文使用场景：

你每周都要让 AI 帮你整理会议纪要，要求它：

- 先提炼结论
- 再列行动项
- 最后按你团队的习惯输出

如果这件事你每次都要重新解释，那它就是这套课要解决的问题。

## 常见误区

- 误区 1：觉得 skill 是给“高阶玩家”用的，和自己没关系
- 误区 2：以为这套课主要是在教某个产品按钮怎么点
- 误区 3：还没进入正文，就急着把每个英文术语背下来

## 本章你应该带走什么

- 这套课不是在教你“多写一点 prompt”
- 它在教你如何把重复工作升级成可复用的 agent 工作流
- skill 只是入口，背后其实是在讲“稳定方法”这件事

## 本章练习

- 用一句话写下你现在理解的 `skill` 是什么，不要求标准，只要求你自己能看懂
- 回想你最近 7 天最常重复让 AI 做的一件事，看看它是不是已经具备“做成 skill”的苗头

## 这章学完后可以立刻做什么

- 写下你最想用这套课解决的一个重复任务
- 判断它更偏内容整理、数据处理，还是开发协作

## 这章最适合用在哪些场景

- 你刚接触这个主题，还没有整体地图
- 你准备把这套课分享给中文同事或朋友
- 你想先确认这门课值不值得继续学下去
