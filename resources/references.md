# 延伸阅读建议

这一页不直接列固定网址，而是告诉你学完这套课之后，下一步应该继续查什么、怎么查、为什么先查这些。

## 建议优先补的方向

- Agent Skills 官方说明与规范
- Anthropic 官方关于 skills、Claude Code、Agent SDK 的文档
- MCP 相关标准与常见 server 实战
- 开源 skill 仓库的结构案例
- 子代理拆分与多代理协作的案例

## 你可以重点对比的几个问题

- 一个 skill 的名字和描述，为什么会影响触发率
- 为什么好用的 skill 通常不是“写得很长”，而是“结构清楚”
- 为什么很多工作流不是缺能力，而是缺稳定的方法
- skills 和 commands、plugins、workflows 之间边界在哪里

## 学完后建议做的三个练习

1. 把你自己反复使用的一段 prompt，整理成一份 skill 草稿
2. 找一个带文件输入输出的工作流，思考 skill 需要哪些工具支持
3. 选一个你熟悉的场景，比如写周报、做课程提纲、审代码，写出 skill 的 `name`、`description` 和主体步骤

## 推荐使用顺序

- 学完前 3 章后：先补 `Skill / Tool / MCP / Subagent` 的官方定义，避免概念混淆
- 学到 05 章后：开始对照开源 skill 仓库，观察别人怎么写结构
- 学到 06-08 章后：再去看 API、CLI、SDK 的官方文档，会更容易看懂
