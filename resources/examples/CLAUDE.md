# 项目说明示例

这个文件示范的是项目级长期上下文应该写什么。

## 项目目标

- 这是一个命令行待办应用
- 重点是保持命令清晰、输出稳定、测试完整

## 技术栈

- Node.js
- TypeScript
- Vitest

## 目录约定

- `src/commands/`：命令实现
- `src/lib/`：通用工具
- `tests/`：测试

## 协作规则

- 新增命令前先检查现有命令命名风格
- 变更 CLI 输出时同步更新测试
- 保持帮助信息简洁一致

## 不该写进这里的内容

- 某一类任务的详细执行 SOP
- 某个特定审查流程的完整步骤

这些更适合写成独立 skill。
