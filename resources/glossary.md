# 术语表

这一页不是为了追求“定义得多准确”，而是为了让你在第一次遇到这些词时，能先抓住一个足够好用的中文理解框架。

## Agent

可以理解为“能根据目标自己调用工具、拆分任务并执行的 AI 助手”。

## Skill

一组可复用的说明和资源，告诉 agent 在某类任务里“应该怎么做”。它不是单条 prompt，而是可打包、可复用、可扩展的一整套工作说明。

## SKILL.md

一个 skill 的核心入口文件。通常会包含：

- skill 名称
- skill 描述
- 主指令
- 对其他文件、脚本、资源的引用

## Progressive Disclosure

渐进式披露。意思是：skill 不会在一开始把所有内容都塞进模型上下文里，而是先暴露“名字和描述”，只有在匹配到任务时才进一步加载正文、参考文件和资源。

## Tool

工具。负责“做事”的具体能力，比如读文件、执行命令、联网搜索、生成图表。

## MCP

`Model Context Protocol`。可以理解为一种把 agent 接到外部数据和系统上的标准方式。它解决的是“怎么连上外部世界”。

## Subagent

子代理。主 agent 把一个子任务分给另一个独立上下文的 agent 去做。它解决的是“怎么拆分和隔离任务”。

## Claude.ai / Claude Desktop

Anthropic 提供的用户界面产品。很多技能相关能力在这些产品里已经被封装好，使用门槛相对低。

## Claude API

面向开发者的接口。你要自己提供文件系统、执行环境、工具接入等能力。

## Claude Code

偏开发场景的 CLI/代码助手环境，适合在项目里结合技能做代码生成、审查、测试等工作。

## Claude Agent SDK

用于构建自定义 agent 应用的开发工具。适合需要主代理、子代理、工具和技能一起协作的场景。

## 怎么读这份术语表

- 如果你第一次看到这些词，不需要强行背定义，先抓住“它主要解决什么问题”
- 如果你已经懂一点 agent 系统，可以把它当成一份中文边界对照表
- 如果你在正文里读到某个词还是模糊，建议回到对应章节再看一次上下文
