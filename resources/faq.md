# 常见问题

这一页重点解决的，不是特别技术化的问题，而是中文初学者最容易在前几章就卡住的理解障碍。

## 1. Skill 不就是一段高级 Prompt 吗？

不完全是。

更准确地说，skill 可以把一整套工作方法打包起来。它不仅有说明文字，还可以引用脚本、模板、图片、额外文档。你可以把它理解成“可复用的工作流包”，而不是“一句更长的提示词”。

## 2. 什么时候该写 skill，什么时候直接写 prompt 就够了？

如果你只是偶尔做一次任务，直接写 prompt 往往更快。

如果你发现自己反复在做同一种流程，比如固定格式的文档分析、固定标准的代码审查、固定结构的数据处理，那么就很适合把它做成 skill。

## 3. MCP 和 skill 到底谁更重要？

它们解决的问题不同，不能简单比较谁更重要。

- `MCP` 解决“接入什么数据和系统”
- `Skill` 解决“拿到这些数据后应该怎么做”

很多时候，二者是一起工作的。

## 4. Subagent 和 skill 会不会重复？

不会。

- `Subagent` 是分工机制
- `Skill` 是方法和知识

你完全可以让一个子代理在自己的上下文里使用某个 skill。

## 5. 我不会写代码，还适合学这门课吗？

适合。

如果你的目标是理解 agent 工作流、知道 skill 的设计思路、能和开发者协作定义流程，这套课依然有价值。只是到了 `API`、`Claude Code`、`Agent SDK` 部分，动手门槛会更高。

## 6. 为什么这套课里经常提到 Claude？

因为原课程就是围绕 Anthropic 的产品体系来讲的。

但 `Agent Skills` 本身被描述为开放标准，所以很多概念并不只能在 Claude 生态里理解。学习这套课时，你可以把 Claude 相关内容看成一组具体案例。

## 7. 我是不是学完就能立刻做出很强的 agent？

不一定。

这套课更像是“把工作流做稳定”的入门课。它能帮助你跨过几个关键门槛：

- 知道 skill 是什么
- 知道 skill 怎么写得更像一个产品化工作流
- 知道它和工具、MCP、子代理如何协作

真正做强 agent，还需要工程实现、调试经验和场景打磨。

## 怎么使用这一页

- 如果你刚开始学，建议遇到卡点就回来查，不要硬顶着往下读
- 如果你已经学完一遍，可以把这页当成复盘提纲，看自己还有哪些概念没真正说清
- 如果你准备把课分享给别人，这页也适合作为“先回答常见疑问”的入口
