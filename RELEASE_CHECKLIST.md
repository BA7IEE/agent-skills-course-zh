# Release Checklist

## 内容检查

- [ ] README 首页信息是否足够清楚
- [ ] 每一章标题是否统一、可读
- [ ] 术语表和 FAQ 是否覆盖常见理解断点
- [ ] 课程映射表是否完整

## 结构检查

- [ ] `lessons/` 下章节顺序正确
- [ ] `resources/` 下辅助文档齐全
- [ ] 仓库内主要链接均可正常跳转

## 发布材料检查

- [ ] `NOTICE.md` 已确认
- [ ] `LICENSE` 已确认
- [ ] `CONTRIBUTING.md` 已确认
- [ ] 是否需要补 `assets/` 中的封面图或分享图

## 法律与边界检查

- [ ] 是否需要在仓库首页增加更明确的出处或引用说明
- [ ] 是否确认当前仓库不再保留第三方原始资料副本

## GitHub 首发检查

- [ ] 仓库名称是否最终确定
- [ ] 仓库简介是否准备好
- [ ] `GITHUB_LAUNCH.md` 中的首发文案是否已确认
- [ ] 首发 commit 信息是否准备好
- [ ] 是否需要首发版本标签
